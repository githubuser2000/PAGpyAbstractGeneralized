# 関係的貨幣経済：数字ではない通貨を基礎にした完全な経済システム

## 1. 基本理念：数字ではない通貨

ここではこのシステムを次のように呼ぶ。

# **関係的貨幣経済**

略して：

```text
RME
```

この経済の通貨は「100ユーロ」のような数字ではなく、**取引可能な関係表**である。

つまり、次のようなものではない。

```text
私は100を持っている。
```

そうではなく、次のようなものである。

```text
私は、権利、義務、請求権、リスク、相手方、時間的拘束、資源との結びつき、能力、担保、社会的費用の束を持っている。
```

したがって、貨幣は一次元の金額ではなく、**構造化された市場状態**として理解される。

基本単位は硬貨でも紙幣でも口座残高でもなく、**関係表**である。

| 関係ID | 主体 | 対象 | 関係タイプ | 請求権 | 義務 | 期間 | 条件 | 移転可能性 | リスク | 法的状態 | 証明 |
|---|---|---|---|---|---|---|---|---|---|---|---|
| R-001 | 個人A | 企業B | 労働 | 熟練労働20時間 | 支払い/保護 | 6月 | 契約有効 | 限定的 | 中 | 有効 | 署名済み |
| R-002 | 企業B | 機械C | 使用 | 機械時間100時間 | 保守 | 第3四半期 | 工場内のみ | はい | 低 | 有効 | 検証済み |
| R-003 | 国家 | 市民A | 税請求 | 納付 | インフラ | 毎年 | 所得/消費 | いいえ | 低 | 法定 | 登録簿 |
| R-004 | 個人D | 個人A | 債務 | 返済 | 支払義務 | 6か月 | 期限到来 | はい | 高 | 争いあり | 契約 |

この表は単なる記録ではない。それ自体が価値の担い手である。

支払いはもはや次のようなものではない。

```text
AがBに500を支払う。
```

そうではなく、次のようになる。

```text
AがBに有効な関係の束を移転する。
- 引渡し請求権
- 労働給付請求権
- 第三者に対する債権
- 資源使用権
- 税額控除
- 保険カバー
- 評判保証
```

これが核心である。

## 2. 通常の市場経済と何が違うのか

通常の市場経済は、ほとんどすべてを価格に圧縮する。

```text
商品 → 価格
労働 → 賃金
リスク → 利子
損害 → 費用
資源 → 市場価値
```

これは単純だから強い。

しかし同時に盲目でもある。数字は、あるものが**なぜ**価値を持つのか、**誰に**何が負われているのか、**どのようなリスク**が付いているのか、**どのような社会的費用**が生じるのか、**どのような依存関係**が作られるのか、そして一緒に売られる**権力関係**が何なのかを示さない。

関係的貨幣経済はこう言う。

```text
価格は情報として貧しすぎる。
市場に必要なのは数字だけではない。
市場には構造化された関係が必要である。
```

したがって、このシステムは特定の意味で市場経済より強い。

```text
市場経済は価格を取引する。
関係的貨幣経済は構造化された現実を取引する。
```

しかし、危険もまさにそこにある。

すべてが関係として記録され、取引可能にされるなら、より効率的に取引されるのは財だけではない。依存、履歴、労働力、社会的結びつき、債務、アクセス、支配、人間の脆弱性も取引可能になる。

それは巨大な経済ブームを生む。同時に巨大な道徳的脅威も生む。

## 3. 中心命題

RMEは次の命題によって機能する。

```text
経済的なものはすべて関係である。
```

価値を持つのは物だけではない。関係も価値を持つ。

```text
人が物を所有する。
人が給付を負う。
企業が労働力を必要とする。
国家が税を徴収する。
顧客が提供者を信頼する。
機械が製品を生む。
土地が収益を生む。
データが予測を生む。
損害が責任を生む。
リスクが保険需要を生む。
希少性が交換力を生む。
```

古典的市場経済では、こうした関係はしばしば隠され、最後には一つの価格に押し込まれる。

RMEではそれらは可視のまま残り、自ら通貨になる。

## 4. 通貨：関係的価値束

ここで本当の通貨は次のように呼ぶ。

```text
関係的価値束
```

略して：

```text
RVB
```

RVBは取引可能な関係からなる表である。

それは単なる「価値」ではなく、価値の構造を含む。

RVBには次のようなものが含まれうる。

| 分類 | 例 |
|---|---|
| 所有関係 | Aが機械Xを所有する |
| 使用権 | Bが倉庫Yを30日間使用できる |
| 労働請求権 | Cが10時間の労働を負う |
| 引渡し請求権 | Dが原材料500単位を引き渡す |
| 債権 | EがFに返済を負う |
| 保険関係 | GがリスクHをカバーする |
| 税関係 | 国家が請求権を納付として認める |
| 評判関係 | 監査者が品質を確認する |
| 責任関係 | 原因者が損害を負担する |
| データ関係 | データセットが分析/利用を許可する |
| 環境関係 | 活動が排出または補償を生む |
| 社会関係 | 依存、介護義務、保護義務 |

重要な点は次である。

```text
通貨は数字ではない。
通貨は関係の束である。
```

数字はセルの中に入ることがある。例えば時間、数量、期限、リスク値である。しかし通貨そのものはその数字ではない。通貨は表全体である。

## 5. すべての通貨表の基本構造

すべての通貨表には少なくとも次の列が必要である。

| 列 | 意味 |
|---|---|
| 主体 | 誰が請求権または義務を持つのか |
| 相手方 | 関係は誰に向けられるのか |
| 対象 | 関係は何を対象とするのか |
| 関係タイプ | 所有、労働、債務、使用、責任、引渡し等 |
| 請求権 | 何を要求できるのか |
| 義務 | 何を履行しなければならないのか |
| 期間 | いつ有効なのか |
| 条件 | どの条件で有効なのか |
| 移転可能性 | 関係を転売できるのか |
| 分割可能性 | 分割できるのか |
| 強制可能性 | 法的に強制できるのか |
| リスク | 不履行、詐欺、紛争の可能性はどれほどか |
| 証明 | 誰が関係を確認するのか |
| 順位 | 他の請求権に優先するのか |
| 道徳状態 | 許可、制限、禁止、譲渡不能 |
| 公開性 | 私的、半公開、国家、匿名化 |
| 削除可能性 | 履行、満了、撤回、死亡等で終了 |

こうして各「貨幣単位」は、小さな契約、小さな資産登録、小さなリスクプロファイルになる。

## 6. このシステムでの支払い

通常の経済では次のようになる。

```text
AがBから商品を買う。
Aが100ユーロを支払う。
Bが商品を引き渡す。
```

関係的貨幣経済では次のようになる。

```text
AがBから商品を買う。
AがBに適切な関係束を移転する。
Bはそれが自分の必要に合うので受け入れる。
```

例：

Bが自転車を売る。Bは「300ユーロ」を求めるのではなく、次を求める。

```text
Bが受け入れるもの:
- 5時間の修理労働に対する有効な請求権
- 国家に認められた税関係
- Cに対する移転可能な請求権
- 不履行に対する保証
```

Aは数字ではなく、構造化された表で支払う。

| 関係 | 内容 | Bが受け入れる理由 |
|---|---|---|
| 労働請求権 | Aが機械を5時間修理する | Bは修理を必要とする |
| 税関係 | 控除可能な公的請求権 | Bは税負担を下げられる |
| 請求権 | Cが引渡しを負う | Bはそれを転売できる |
| 保証 | 不履行がカバーされる | リスクが減る |

価格はもはや数字ではない。価格は**受容パターン**である。

```text
価格とは売り手が受け入れる表である。
支払いとは買い手が移転する表である。
市場とは適合する表を結びつけるシステムである。
```

## 7. 価値はどのように生まれるか

価値は単に行を数えることで生まれるわけではない。

無価値、危険、または禁止された関係が1,000個ある表は、非常に強い関係が3個ある表より価値が低いかもしれない。

価値は次のものから生まれる。

```text
有用性
希少性
強制可能性
信頼
移転可能性
時間的近さ
リスク内容
社会的受容
国家による承認
生産性
法的明確性
```

多くの市場参加者が受け入れる関係束は強い。

強制が難しく、道徳的に重く、危険で、流動性が低く、または法的に禁止されている関係束は弱い。

## 8. 数字の役割

RMEは数字を禁止しない。ただ数字の地位を下げる。

数字はもはや通貨そのものではない。数字は表の内部にある補助値にすぎない。

例：

```text
10時間
3か月
50キロワット時
2ヘクタール
5%の不履行リスク
CO₂ 40トン
8回の引渡し
```

しかし価値は単独の数字ではなく関係にある。

```text
誰の10時間なのか。
誰のためなのか。
どの仕事なのか。
どの条件下なのか。
どの品質なのか。
どのリスクなのか。
どの法的保護があるのか。
移転可能なのか。
自発的なのか強制なのか。
```

これが違いである。

市場経済は問う。

```text
いくらか。
```

RMEは問う。

```text
どの関係が移転されるのか。
どの権力が移転されるのか。
どの義務が移転されるのか。
どの未来が移転されるのか。
どの依存が作られるのか。
```

それがRMEを強くする。同時に危険にもする。

所有は単なる「占有」ではなく、関係表である。

住宅は住宅 + 所有者だけで構成されるのではなく、次で構成される。

```text
所有権
使用権
抵当権
納税義務
近隣義務
維持管理義務
保険関係
エネルギー消費
環境要件
相続権
賃借権
国家登録
```

古典的市場経済では、住宅はたとえば次のように現れる。

```text
住宅価値: 600,000ユーロ
```

RMEでは次のように現れる。

```text
権利、義務、リスク、収益、費用、拘束、アクセス可能性の束。
```

こちらの方が現実に近い。

住宅は単なる物ではない。権利と義務のネットワークにおけるノードである。

## 10. RMEにおける企業

企業は単なる貸借対照表合計を持つ法人ではない。企業は圧縮された関係ノードである。

企業は次で構成される。

```text
労働関係
顧客関係
サプライチェーン関係
所有関係
債務関係
評判関係
知識関係
データ関係
機械関係
許認可関係
責任関係
税関係
```

企業の関係が次のようであれば、その企業は豊かである。

```text
生産的
承認されている
流動的
法的に安定している
希少である
信頼できる
多方面に接続可能である
```

企業の関係が次のようであれば、その企業は弱い。

```text
争いがある
危険である
依存的である
短期的である
法的に不確実である
道徳的に有害である
移転できない
```

RMEにおける利益は単に次を意味しない。

```text
収入 - 費用。
```

利益は次を意味する。

```text
企業が生産後に以前より良い関係束を生み出した。
```

つまり：

```text
より多く受け入れられる請求権
より少ない有害な義務
より良い納品関係
より良い評判
より高い強制可能性
より低いリスク
より多くの未来選択肢
```

これは金銭的利益より広い。

## 11. RMEにおける労働

ここに決定的な道徳的対立がある。

RMEの正当なバージョンでは、人間が取引されるのではなく、自発的で限定された労働関係だけが取引される。

許容されるのは次のようなものだ。

```text
個人Aが自発的に、
企業Bのために、
期間Tに、
条件Cの下で、
特定の給付Lを行うことを約束し、
解約権、
保護権、
賃金請求権、
労働時間上限、
本人の譲渡不能性を持つ。
```

これが労働関係である。

表として表すと次のようになる。

| 個人 | 雇用者 | 給付 | 期間 | 義務 | 反対給付 | 解約 | 保護権 | 移転可能性 |
|---|---|---|---|---|---|---|---|---|
| A | 企業B | 熟練労働 | 3か月 | 労働 | 報酬/保険 | はい | はい | 制限あり |

重要な境界は次である。

```text
労働給付は契約対象になりうる。
人間自身は決してそうであってはならない。
```

この境界がなければ、システムは転倒する。

資本になりうるもの：

```text
機械使用権
データアクセス
顧客信頼
ライセンス
特許
労働約束
サプライチェーン
国家許可
安価な信用
社会的評判
保険保護
原材料アクセス
```

これによりRMEは、市場経済がしばしば隠すものを可視化する。

```text
資本とは凝縮された関係権力である。
```

多くの強い関係を持つ者が権力を持つ。

多くのお金を持つ者だけが権力を持つのではない。

## 13. RMEにおける信用

信用は未来関係である。

通常の経済では：

```text
銀行が100,000ユーロの融資を行う。
債務者が後で返済する。
```

RMEでは：

```text
銀行が未来給付、担保、リスク、法的請求権を含む
承認された関係束を作る。
```

信用RVBには次が含まれる。

| 要素 | 意味 |
|---|---|
| 債務者 | 誰が負うのか |
| 債権者 | 誰に負うのか |
| 未来給付 | 何が期待されるのか |
| 担保 | どの関係が責任を負うのか |
| 順位 | 誰が先に弁済を受けるのか |
| リスク | 不履行確率 |
| 目的 | 生産、消費、投機 |
| 支配 | 債権者はどの介入権を得るのか |
| 不履行 | 履行されない時に何が起こるのか |

銀行はこのシステムで**関係変換者**になる。

取引しにくい関係を、より受け入れやすい関係に変える。

例：

```text
多数の不確実な返済約束
→ 束ねられ、検査され、保険化される
→ 取引可能な信用束
```

これは強力である。

しかし金融危機における証券化のように有害にもなりうる。しかもより深い。なぜなら、債務だけでなく生活上のすべての関係が梱包されうるからである。

## 14. 中央銀行と通貨量

通常の中央銀行は通貨量、金利、流動性を制御する。

RMEの中央銀行は次を制御する。

```text
承認
標準化
信頼ランク
公的受容
関係の最低品質
担保クラス
削除規則
移転可能性規則
リスクウェイト
危機処理
```

中央銀行は単に次を問うのではない。

```text
流通しているお金はいくらか。
```

そうではなく：

```text
どの関係が流通しているのか。
それらの品質はどうか。
どれほど有害か。
どれほど入れ子になっているか。
誰が誰に依存しているか。
どの関係が一般的支払手段として認められるか。
どの関係が自由、安定、尊厳を脅かすか。
```

このシステムでインフレーションは単なる価格上昇ではない。

インフレーションとは：

```text
弱く、不安定で、過度に引き延ばされた関係が多すぎるほど価値あるものとして受け入れられている。
```

デフレーションとは：

```text
受け入れられる関係が少なすぎる。
全員が硬く、安全で、国家保証のある関係だけを保持する。
交換が滞る。
```

多くの市場参加者が突然次を認識すると危機が起きる。

```text
私たちが通貨として受け入れていた関係は履行不能である。
```

その時崩壊するのは貨幣への信頼だけではなく、関係への信頼でもある。

## 15. 国家

RMEにおいて国家は単なる規制者ではなく、最も重要な道徳的・法的境界設定者である。

国家は五つのことを行う。

### 1. 譲渡不能な関係を定義する

例：

```text
人間の尊厳
身体の自由
基本権
雇用解約の自由
強制からの保護
債務奴隷からの保護
データ保護
家族基本権
政治的権利
```

これらは取引されてはならない。

### 2. 特定の関係を税務上受け入れ可能にする

国家はこう言える。

```text
私はこれらの関係を公的義務の履行として受け入れる。
```

例：

```text
介護労働
CO₂補償
インフラ貢献
金銭等価物
公共サービス
税請求権
```

これにより特定の関係への需要が生まれる。

### 3. 有害な関係を禁止する

例：

```text
人の取引
無制限の労働拘束
強制契約
解約不能な債務関係
売られた投票権
売られた身体権
無制限のデータ支配
強制された依存関係
```

### 4. 弱い当事者を保護する

関係市場は貨幣市場よりはるかに強力なので、より強い保護権が必要である。

```text
労働法
賃貸法
消費者保護
データ保護
債務者保護
破産法
独占禁止法
差別禁止法
最低基準
```

### 5. 危機を安定化する

関係市場が凍結した時、国家は安全な公的関係を提供しなければならない。

```text
基本供給
公的労働
基本請求権
国家保証支払関係
債務削減
関係削除
有害関係の転換
```

国家がなければ、RMEはすぐに私的支配経済になる。

## 16. RMEにおける市場

市場はもはや価格が形成される場所だけではない。

市場は関係を互換可能にする場所である。

さまざまな市場がある。

| 市場 | 取引されるもの |
|---|---|
| 財市場 | 所有関係と引渡し関係 |
| 労働市場 | 限定された労働給付関係 |
| 資本市場 | 未来請求権、担保、参加権 |
| 信用市場 | 債務・返済関係 |
| 保険市場 | リスク引受け |
| データ市場 | データ使用権 |
| エネルギー市場 | 生産・消費関係 |
| 住宅市場 | 使用・賃貸・所有関係 |
| 評判市場 | 認証、監査、信頼シグナル |
| 環境市場 | 排出、補償、再生 |
| 介護市場 | 介護・支援関係 |
| 国家市場 | 税、ライセンス、許認可関係 |

市場メカニズムは次である。

```text
供給表が需要表と出会う。
清算システムが互換性を検査する。
関係が合えば交換が起こる。
```

したがって価格は中心情報ではない。中心情報は次である。

```text
これらの関係は互いに合うのか。
```

## 17. 競争

競争は単に安くなることではない。

より良い関係を提供することである。

より良いとは次を意味する。

```text
より信頼できる
より速い
より低リスク
より法的に安全
より環境的に清潔
より社会的に受け入れられる
より移転しやすい
より組み合わせやすい
より透明
より搾取的でない
```

企業は単に最安値だから勝つのではなく、その関係がより多くの市場で受け入れられるから勝つ。

これがこのシステムが強い理由の一つである。

```text
古典的市場経済はしばしば価格を最適化する。
RMEは関係の接続可能性を最適化する。
```

## 18. なぜこのシステムは市場経済より強いのか

ここでいう「強い」は、自動的に「良い」または「公正」を意味しない。

強いとは次を意味する。

```text

より多くの制御可能性
より多くの取引可能性
隠れた価値のより大きな動員
依存関係に対するより大きな支配
未来を担保にするより大きな能力
複雑な財を比較するより大きな能力
```

RMEはいくつかの理由で強い。

### 第一に、見えない価値を見えるようにする

市場経済では多くのものがうまく表されない。

```text

```

RMEはこれらを関係として捉えられる。

それによってそれらは取引可能、保険可能、課税可能、規制可能、評価可能になる。

それがブームを生む。

### 第二に、より多くのものを流動化する

今日、多くの資産は取引しにくい。

```text
未来の労働能力
不完全な契約
小額請求権
地域的信頼
使用権
介護サービス
データ権

社会的貢献
```

RMEはそれらを取引可能な束にできる。

以前は潜在的にしか存在しなかったものが市場権力になる。

### 第三に、情報盲を減らす

価格は言う。

```text
これは100である。
```

関係表は言う。

```text
これは有用だが危険である。
未解決の責任がある。
環境的負荷がある。
三つの供給者に依存している。
国家が認めている。
二週間で移転できる。
一部しか保険されていない。
```

これははるかに豊かな市場情報である。

### 第四に、外部費用を組み込める

環境費用、介護費用、社会的損害、リスクは価格から簡単に消えない。

それらは通貨そのものの中で関係として立つ。

製品が実際には大きな損害を生んでいるなら、単に安いとは言えない。その損害は負の関係として製品に付く。

### 第五に、生産をより精密に調整できる

企業は価格だけでなく関係ネットワークを見るため、生産をより正確に計画できる。

経済は見る。

```text
どこで労働力が不足しているか
どこで供給網が壊れているか
どこでリスクが上がっているか
どこで資源が使われていないか
どこで信頼が不足しているか
どこで国家許可が妨げているか
どこで介護需要が伸びているか
どこで環境負荷が生じているか
```

これは主に価格でコミュニケーションする市場より強い。

## 22. 悪いバージョンにおける雇用者と労働者

正当なバージョンでは労働契約が取引される。

悪いバージョンでは、人間がその依存関係を通じて事実上取引される。

それは露骨な「人間の売買」ではなく、次のような関係束によって行われる。

```text
独占的労働拘束
債務拘束
住居拘束
ビザ拘束
評判統制
データ統制
解約罰金
労働請求権の再販売
将来所得の統制
```

すると技術的には契約に見えても、新しい隷属が生じる。

例：

```text
労働者Aは企業Bに労働給付を負っている。
企業Bはこの請求権を企業Cに売る。
Aは形式的には売られていない。
しかしAの未来への支配は移動する。
```

ここが危険な点である。

人間は直接売られていないが、その人生を決定する重要な関係が売られている。

十分な関係が移転されれば、違いは実質的には法的形式だけになる。

```text
人の労働時間、住居、債務、居住権、評判、データを支配する者は、
その人を支配する。
```

したがって、次のように言うだけでは不十分である。

```text
私たちは関係を取引しているだけで、人間を取引しているわけではない。
```

人間は自分の関係を通じて完全に支配されうる。

## 23. なぜ政府が不可欠なのか

純粋に私的なRMEは道徳的に脱線する。

それは全員が悪人だからではなく、市場が利用可能なすべての関係を発見するからである。

ある関係が利益を生むなら、それを取引可能にする圧力が生まれる。

したがって国家には硬い境界が必要である。

```text
この関係は売れない。
この関係は担保にできない。
この関係は移転できない。
この関係は永久的に拘束できない。
この関係は基本権に反して使えない。
この関係は濫用時に消滅する。
```

このような境界がなければ、依存の市場が生じる。

国家は市場失敗を修正するだけでなく、**市場の越境**を防がなければならない。

これは規制以上のものだ。文明の保護である。

これは大きな利点である。

しかしここにも悪い面がある。

```text
豊かな主体が清潔な関係を買う。
```

一部の環境損害は取引可能であってはならない。

## 35. 教育と能力

教育はRMEにおいて中心的な資本形態になる。

能力は物ではなく関係である。

```text
人は条件Yの下で品質Zにより課題Xを遂行できる。
```

これは認証されうる。

利点：

```text
能力がより正確に見える。
人はより良い仕事を見つけられる。
企業はより適合した採用ができる。
継続教育がより資金調達しやすくなる。
```

危険：

```text
人が能力プロファイルへ縮減される。
過去の失敗が表に残る。
認証されていない能力が見えなくなる。
教育が全面的利用機械になる。
```

したがって次が必要である。

```text
再評価権
忘れられる権利
公教育
差別的能力プロファイルの禁止
```

## 36. 民主主義

政治的関係が取引可能になると、RMEは民主主義を脅かす。

危険なのは次の市場である。

```text
投票権
政治的忠誠
抗議放棄
情報アクセス
世論への影響
政党アクセス
市民権
居住権
```

したがって次が成立しなければならない。

```text
基本的な政治関係は譲渡不能である。
```

人は自分の票を売ってはならない。

企業は関係的支配によって政治的依存を作ってはならない。

国家は経済関係が民主的関係を飲み込むことを防がなければならない。

## 37. RMEが市場経済より完全に見える理由

市場経済は交換と価格形成に強い。

しかし次については弱い。

```text
権力
依存
リスク
生態
データ
未来義務
社会的再生産
信頼
介護
政治的影響
```

RMEはこれらすべてを同じシステムで表せる。

したがってRMEは単なる市場経済より完全である。財と価格だけでなく次を扱うからである。

```text
主体
権利
義務
時間
リスク
信頼
権力
データ
国家
生態
道徳的境界
```

それは経済システムであると同時に、法、データ、権力秩序のシステムである。

まさにそのため中立ではない。

## 38. RMEの基本法

良いRMEには最高原則が必要である。

```text
経済的なものはすべて関係的に記録されうる。
しかし関係的なものすべてが経済的に取引されてよいわけではない。
```

これが最も重要な文である。

これがなければシステムは悪くなる。

これがあれば、全体主義にならずに強くなれる。

## 39. システムの良いバージョン

良いバージョンは次のようになる。

```text
通貨 = 取引可能な関係束
人間 = 譲渡不能な人格
市場 = 権利、義務、資源のマッチング
国家 = 境界設定者かつ安定保証者
福祉国家 = 安全な基本関係の提供者
企業 = 生産的関係ノード
銀行 = 関係変換者
中央銀行 = 品質と承認の機関
法 = 有害な取引可能性からの保護
```

このバージョンは、より多くの現実を市場へ持ち込むため、市場経済より強い。

しかし硬い非市場領域によって人間の尊厳を守る。

## 40. システムの悪いバージョン

悪いバージョンは次のようになる。

```text
通貨 = すべての関係
人間 = 利用可能なポートフォリオ
労働 = 取引可能な拘束
債務 = 支配手段
データ = 人格アクセス
評判 = 規律権力
国家 = 弱い、または買収されている
企業 = 私的支配ノード
市場 = 全面利用
```

このバージョンはおそらく残酷なブームを生む。

なぜか。

すべてを動員するからである。

```text
あらゆる能力
あらゆる債務
あらゆる弱さ
あらゆる拘束
あらゆる未来
あらゆる親密性
あらゆる依存
あらゆる社会関係
```

しかしこのブームは道徳的に毒されている。

それは人間さえ燃料にする機械のように強い。

## 41. 特別な経済ブーム

このブームは五つの理由で特に強い。

### 1. より多くの担保

人と企業は、より多くのものを担保として使える。

```text
未来収入
労働約束
データ権
供給網
評判
保険
使用権
```

これにより信用能力が爆発する。

### 2. より多くの流動性

以前は固定されていたものが取引可能になる。

```text
小額請求権
時間枠
地域請求権
マイクロサービス
不完全な権利
```

### 3. より多くのマッチング

供給と需要がより正確に結びつく。

単に次ではない。

```text
仕事が必要だ。
```

そうではなく：

```text
この技能が必要で、
この期間、
このリスクの下、
この保険付きで、
この地域で、
この法的品質を持って必要である。
```

### 4. より多くの支配

企業はプロセスをはるかに正確に制御できる。

これは効率を高める。

### 5. 未来のより大きな利用

未来がより取引可能になる。

未来の労働、未来の納品、未来のデータ、未来の支払い、未来の使用が、今日すでに経済的に利用される。

これは成長を生む。

しかし人と社会を未来へ向けて債務化することもある。

## 42. ブームの負の代価

ブームの代価は次である。

```text
より多くの監視
より多くの契約依存
より大きな複雑性
プラットフォームのより大きな権力
自己利用へのより大きな圧力
債務隷属のより大きなリスク
より少ない私的生活空間
より少ない自発的自由
評価されない余地の減少
```

古典的市場経済では、人は貧しくても人生全体が自動的に取引可能な表として見えるわけではない。

RMEでは次が脅威となる。

```text
すべての弱さが関係になる。
すべての関係がリスクになる。
すべてのリスクが価格になる。
すべての価格が市場障壁になる。
```

これは残酷になりうる。

## 43. 例：危険な生活データセット

悪いRMEでは、ある人は次のように現れるかもしれない。

| 領域 | 関係 | 市場効果 |
|---|---|---|
| 労働 | 頻繁な転職 | 信頼低下 |
| 健康 | 慢性リスク | 保険費用上昇 |
| 債務 | 古い義務 | 信用関係悪化 |
| 家族 | 介護義務 | 可用性低下 |
| 居住地 | 弱いインフラ | 労働接続性低下 |
| データ | 否定的消費プロファイル | 悪いオファー |
| 教育 | 証明書不足 | 低い分類 |
| 評判 | 雇用者との紛争 | 市場排除 |

このシステムは非常に情報豊かである。

しかし人を固定化することもある。

したがって次の権利が必要である。

```text
新しい出発
削除
文脈分離
異議申立て
人間による審査
非差別
```

## 44. 悪い側面をどう抑えるか

それを魔法のように消すことはできない。制度的に飼いならすことしかできない。

必要なのは次である。

| 問題 | 対策 |
|---|---|
| 人間関係の取引 | 譲渡不能性の法 |
| 労働隷属 | 解約権、最大拘束期間、最低保護 |
| 債務奴隷 | 破産、債務削減、利子上限 |
| データ支配 | データ保護、削除、目的制限 |
| プラットフォーム独占 | 独占禁止、相互運用性 |
| 評判の牢獄 | 再評価権 |
| 社会的排除 | 公的基本関係 |
| 有害な束ね | 解体法 |
| 政治的買収可能性 | 取引可能な市民権の禁止 |
| 環境外部委託 | 取引不能な環境限界 |

国家は市場ルールを設定するだけでなく、どの関係がそもそも市場化可能なのかを決定しなければならない。

## 45. RMEの憲法

完全なRMEには一種の経済憲法が必要である。

それは次のように書ける。

### 第1条：人格保護

```text
人は取引可能な関係運搬者ではない。
自発的で、限定され、撤回可能な給付だけが市場化可能である。
```

### 第2条：譲渡不能性

```text
基本権、身体の自由、政治的権利、核心的アイデンティティは譲渡不能である。
```

### 第3条：透明性

```text
すべての取引可能な関係は、起源、リスク、義務、相手方を開示しなければならない。
```

### 第4条：削除

```text
履行済み、濫用された、または古くなった関係は削除または訂正可能でなければならない。
```

### 第5条：解体

```text
いかなる主体も、ある人の労働、住居、信用、データ、保険、居住権を完全に支配してはならない。
```

### 第6条：国家的基本関係

```text
すべての人は、生命、法、健康、教育、参加のための最低限安全な関係に対する権利を持つ。
```

### 第7条：市場の限界

```text
人の自由を構造的に破壊するものは通貨になってはならない。
```

## 46. 会計

RMEにおける会計は、もはや金額による複式簿記だけではない。

それは関係的な貸借評価である。

家計や企業は次を持つ。

```text
能動的関係:
- 請求権
- 権利
- 担保
- 使用権
- 債権
- 保証

受動的関係:
- 義務
- 債務
- 責任
- リスク
- 依存
- 環境負担
```

貸借とは次を問うことである。

```text
どの関係が私を強くするか。
どの関係が私に負担をかけるか。
どの関係が移転可能か。
どの関係が危険か。
どの関係がまもなく期限切れか。
どの関係が争われているか。
```

企業は豊かに見えても、多くのリスクと依存を負っていれば関係的に弱いことがある。

別の企業はお金が少なくても、安定し、生産的で、信頼できる関係を持つなら関係的に強いことがある。

## 47. 成長の測定

国内総生産だけでは不十分である。

RMEは成長を、国家の関係ストックの改善として測る。

重要な指標は次のようなものだ。

```text
生産的関係
安定した労働関係
健全な信用関係
低い有害依存
生態的再生関係
教育関係
信頼密度
社会保障
イノベーション関係
供給網安定性
```

社会は、より多く売る時だけ成長するのではない。

関係がより持続可能で、自由で、生産的で、有害でなくなる時に成長する。

## 48. 危機

RMEの危機にはさまざまな形がある。

### 信頼危機

どの関係が有効なのか誰にも分からなくなる。

### 履行危機

多くの主体が義務を履行できない。

### データ危機

証明が操作または盗難される。

### 道徳危機

あまりに多くの人が取引可能な関係によって貶められる。

### 流動性危機

すべての人が国家保証関係だけを求める。

### 権力危機

一部のプラットフォームが中心的関係ノードを支配する。

その時、国家は次を行わなければならない。

```text
関係を凍結する
有害な束を分割する
公的保証を与える
債務を削除する
最低供給を確保する
独占を解体する
データを訂正する
法的明確性を作る
```

## 49. 通常の市場経済がそれに比べ弱い理由

通常の市場経済は強く単純化するため弱い。

それは言う。

```text
すべてには価格がある。
```

RMEは言う。

```text
すべてには構造がある。
```

通常の市場経済はしばしば遅れて認識する。

```text
供給網が壊れる。
労働が搾取される。
環境が破壊される。
データ権力が成長する。
債務が有害になる。
人々が依存的になる。
```

RMEはこれらをより早く可視化できる。なぜならそれらは通貨そのものの中に関係として現れるからである。

これはRMEを分析的、組織的、金融技術的に強くする。

しかしその代価は、システムが社会の中へより深く入り込むことである。

## 50. 決定的矛盾

RMEはすべてを可視化したがる。

しかし自由には、完全には見えず、完全には取引可能ではない領域が必要である。

そこから基本的対立が生まれる。

```text
システムが経済的に完全であるほど、
人間的には危険になる。
```

完全な関係市場機械はおそらく非人間的である。

人間的なRMEはしたがって不完全でなければならない。

それは意識的に盲点を許さなければならない。

```text
プライバシー
非評価
非移転性
赦し
新しい出発
無償の尊厳
売れない権利
```

これは弱さではない。正当性の条件である。

## 52. 結論

関係的貨幣経済は、市場を価格へ縮減しないため、市場経済より強い。

それはより多くを見ることができる。

```text
権利
義務
リスク
依存
信頼
未来
データ
環境
権力
```

それはより多くを取引できる。

```text
請求権
使用
担保
労働給付
保険
データ権
供給網
評判
生態的義務
```

それはより多くを制御できる。

```text
生産
信用
リスク
社会政策
環境損害
労働市場
公共財
```

だから特別な経済ブームを生む。

しかしこのブームには暗い裏面がある。

```text
見えるようになったものはすべて評価されうる。
評価されたものはすべて取引されうる。
取引されたものはすべて人間に対する権力になりうる。
```

したがって中心的な政治課題は、このシステムを効率化することだけではない。

中心課題は次である。

```text
どの関係が決して通貨になってはならないのかを決めること。
```

RMEの最良のバージョンは、通常の市場経済より複雑で、正直で、生産的な経済である。

最悪のバージョンは、人々が公式には売られないが、実際には労働、債務、データ、住居、評判、依存関係によって支配される全面市場である。

両者の違いは技術ではない。

違いは法である。

さらに深く言えば：

```text
違いは、
市場が人間に奉仕するのか、
それとも人間が関係として市場に属するのか、
という決定である。
```
